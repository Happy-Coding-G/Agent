正则表达式使用方法

```
1. 普通字符
普通字符在正则表达式中匹配它们自身。例如：
- `a` 匹配字符 "a"
- `hello` 匹配字符串 "hello"

2. 元字符（特殊字符）
元字符是具有特殊含义的字符，通常用于构建复杂的模式：
- `.`（点）：匹配除换行符之外的任何单个字符。
  - 例子：`a.b` 可以匹配 "a+b", "a-b", "a1b" 等。
- `^`：匹配字符串的开头。
  - 例子：`^abc` 匹配 "abc" 在字符串开头的情况。
- `$`：匹配字符串的结尾。
  - 例子：`abc$` 匹配 "abc" 在字符串末尾的情况。
- `*`：匹配前面的字符零次或多次。
  - 例子：`ab*` 匹配 "a", "ab", "abb", "abbbbb" 等。
- `+`：匹配前面的字符一次或多次。
  - 例子：`ab+` 匹配 "ab", "abb", "abbbbb"，但不匹配 "a"。
- `?`：匹配前面的字符零次或一次（非贪婪匹配）。
  - 例子：`ab?` 匹配 "a" 或 "ab"。
- `|`：表示 "或"（OR）操作。
  - 例子：`a|b` 匹配 "a" 或 "b"。

3. 字符集（字符类）
字符集用于匹配字符集合中的任何一个字符：
- `[abc]`：匹配 "a"、"b" 或 "c"。
- `[a-z]`：匹配小写字母从 "a" 到 "z" 的任意字符。
- `[^abc]`：匹配任何不在 "a"、"b"、"c" 中的字符，`^` 表示取反。
- `[0-9]`：匹配数字 0 到 9 的任意字符。
- `\d`：匹配任何数字字符，相当于 `[0-9]`。
- `\D`：匹配任何非数字字符，相当于 `[^0-9]`。
- `\w`：匹配任何单词字符（字母、数字或下划线），相当于 `[A-Za-z0-9_]`。
- `\W`：匹配任何非单词字符，相当于 `[^A-Za-z0-9_]`。
- `\s`：匹配任何空白字符（包括空格、制表符、换行符等）。
- `\S`：匹配任何非空白字符。

4. 量词
量词用于指定前面的元素可以出现的次数：
- `{n}`：匹配前面的元素恰好出现 n 次。
  - 例子：`a{3}` 匹配 "aaa"。
- `{n,}`：匹配前面的元素至少出现 n 次。
  - 例子：`a{2,}` 匹配 "aa"、"aaa"、"aaaa" 等。
- `{n,m}`：匹配前面的元素至少出现 n 次，至多出现 m 次。
  - 例子：`a{1,3}` 匹配 "a"、"aa"、"aaa"。

5. 分组和捕获
正则表达式可以使用圆括号 `()` 来定义子表达式，并捕获匹配的内容：
- `(abc)`：将 "abc" 作为一个整体进行匹配，并捕获它。
- `(?:abc)`：匹配 "abc" 但不进行捕获（非捕获组）。
- 捕获的内容可以通过序号来引用：`\1`, `\2` 等。

6. 反向引用
- `\1`, `\2` 等：引用前面的捕获组。例如，`(a)(b)\1\2` 可以匹配 "abab"。

7. 贪婪与非贪婪匹配
- 默认情况下，量词是贪婪的，会尽可能多地匹配字符。
  - 例子：`a.*b` 在字符串 "a123b456b" 中会匹配 "a123b456b"（匹配到最后一个 "b"）。
- 非贪婪匹配：在量词后加上 `?` 可以实现非贪婪匹配，它会尽可能少地匹配字符。
  - 例子：`a.*?b` 在字符串 "a123b456b" 中会匹配 "a123b"（匹配到第一个 "b"）。

8. 转义字符
由于一些元字符具有特殊含义，如果需要匹配它们的字面含义，必须使用反斜杠进行转义：
- `\.`：匹配字面上的点（"."）。
- `\\`：匹配反斜杠。

9. 常见预定义字符类
- `\b`：匹配单词边界。
  - 例子：`\bword\b` 匹配完整的 "word" 但不会匹配 "sword" 或 "words"。
- `\B`：匹配非单词边界。
  - 例子：`\Bword\B` 匹配 "swording" 中的 "word"。

10. 注释
正则表达式支持添加注释，帮助提升可读性：
- `(?#...)`：表示注释，正则解析时会忽略注释部分。
  - 例子：`a(?#这是一个注释)b` 匹配 "ab"。

11. 零宽断言
零宽断言用于匹配前后的位置，但不消耗字符：
- `(?=...)`：正向先行断言，匹配表达式之前的位置。
  - 例子：`a(?=b)` 匹配 "a" 仅当它后面跟着 "b"，不会匹配 "ab" 以外的 "a"。
- `(?!...)`：负向先行断言，匹配不符合表达式的位置。
  - 例子：`a(?!b)` 匹配 "a" 仅当它后面不是 "b"。
- `(?<=...)`：正向后发断言，匹配在表达式之后的位置。
  - 例子：`(?<=a)b` 匹配 "b" 仅当它前面是 "a"。
- `(?<!...)`：负向后发断言，匹配不符合表达式之前的位置。
  - 例子：`(?<!a)b` 匹配 "b" 仅当它前面不是 "a"。

```
