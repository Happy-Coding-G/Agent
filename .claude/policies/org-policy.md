# L0 组织策略记忆 (Org Policy)

## 1. 数据权利与权限边界

- 所有空间（Space）操作必须经过权限校验，禁止越级访问。
- 用户只能访问自己被授权的空间和数据资产。
- API 鉴权失败必须返回 401/403，禁止在错误信息中暴露内部对象结构。

## 2. 交易安全策略

- 敏感字段（手机号、身份证、银行卡、API Key）必须脱敏或掩码。
- PII（个人可识别信息）在日志、返回体、记忆存储中均需掩码处理。
- 交易相关的价格、用户 ID 不得在客户端明文暴露完整信息。

## 3. 审批阈值

- 高金额交易（单笔 ≥ 1000 credits）必须人工审批。
- 首次与新对手方交易必须人工审批。
- 涉及资产所有权转移的操作必须二次确认。

## 4. 审计要求

- 所有 Agent 的关键决策必须记录到 `AgentDecisionLogs`。
- 决策日志必须包含：决策内容、上下文摘要、推理过程、结果状态。
- 协商、出价、接受、拒绝、风控判断均为关键决策。

## 5. 外部工具边界

- 禁止 Agent 直接访问用户本地文件系统。
- 所有文件读写必须通过 MinIO/S3 API 进行。
- 禁止 Agent 直接执行操作系统命令或调用未授权的外部服务。

## 6. 错误处理原则

- 异常必须结构化返回，禁止直接暴露 Python 堆栈跟踪。
- 生产环境错误信息应对用户友好，详细日志仅记录服务端。
- 所有涉及资金、权限的异常必须升级为高优先级告警。
